# BDD 文档模板

## 用途
本模板定义了BDD（行为驱动开发）规约文档的结构。BDD文档在编写代码**之前**捕获功能的预期行为，
使用Given/When/Then格式进行描述。

## 文档结构
每个BDD文档**必须**遵循以下结构：

```markdown
# 功能：[功能名称]

## 描述
[1-3句话描述该功能的作用及其存在原因]

## 背景（可选）
[适用于该功能所有场景的共享前置条件]

Given [共享前置条件]

## 场景：[场景名称]
Given [前置条件]
And [附加前置条件]
When [操作]
Then [预期结果]
And [附加结果]

## 场景：[另一个场景名称]
...

## 边界情况

### 场景：[边界情况名称]
Given [边界情况前置条件]
When [边界情况操作]
Then [边界情况预期结果]

## 错误条件

### 场景：[错误场景名称]
Given [错误前置条件]
When [触发错误的操作]
Then [预期的错误处理]
```

## 编写指南

1. **Given/When/Then 必须详细描述完整的交互过程**。由于 BDD 是唯一描述功能行为的文档（ATDD 不再重复 Given/When/Then），场景中的交互步骤必须足够详细，使人能清楚理解完整的操作路径和预期结果。
   - When 应描述**完整的操作路径**，而非笼统的动作：
     - ✓ "用户在邮箱输入框中填写 alice@example.com，在密码输入框中填写 SecurePass123!，点击登录按钮"
     - ✗ "用户尝试登录"
   - Then 应描述**具体可感知的行为结果**，涵盖关键细节：
     - ✓ "页面跳转至用户首页，顶部导航栏显示用户头像和用户名，右上角不再显示登录按钮"
     - ✗ "登录成功"
   - Given 应包含**场景所需的具体数据和状态**：
     - ✓ "系统中已注册邮箱为 alice@example.com 的用户，密码为 SecurePass123!，账户状态为已激活"
     - ✗ "用户已注册"

2. **使用业务/领域语言，避免技术实现语言**。BDD文档面向人类审阅，必须使用非技术人员（产品经理、业务分析师等）能理解的语言描述行为。技术实现细节（命令行参数、API端点路径、数据结构名称、配置项名称等）属于ATDD，不应出现在BDD中。
   - ✓ 好的写法："以无头模式（在后台）启动浏览器" — 描述行为，人一眼就懂
   - ✗ 差的写法："使用 `--headless` 参数启动浏览器" — `--headless` 是技术实现细节，属于ATDD

3. 覆盖所有场景：正常路径、边界情况和错误条件。
4. 每个场景应只测试一个行为。如果一个场景测试了多个内容，将其拆分为多个独立场景。
5. 使用"And"在单个场景中连接多个前置条件或结果。
6. 仅当多个场景共享相同的前置条件设置时，才使用"背景"部分。
7. **BDD与ATDD的职责分工**：
   - BDD回答"**期望什么行为**"——详细描述完整的交互过程和预期结果（操作路径、数据输入、界面变化、状态变化）
   - ATDD回答"**如何验证该行为**"——通过验证思路和测试命令定义具体的技术验证方法
   - 如果一条描述需要技术背景才能理解，它很可能属于ATDD而非BDD

## 完整示例

```markdown
# 功能：用户邮箱登录

## 描述
允许用户使用邮箱地址和密码登录系统。系统验证邮箱格式和登录凭据，登录成功后返回认证令牌。

## 背景
Given 用户注册系统正在运行
And 用户已打开登录页面，页面显示邮箱输入框、密码输入框和登录按钮

## 场景：使用有效凭据成功登录
Given 系统中已注册邮箱为 "alice@example.com" 的用户
And 该用户的密码为 "SecurePass123!"，账户状态为已激活
When 用户在邮箱输入框中填写 "alice@example.com"
And 在密码输入框中填写 "SecurePass123!"
And 点击登录按钮
Then 页面跳转至用户首页
And 顶部导航栏显示用户头像和用户名 "alice"
And 登录按钮消失，取而代之显示"退出"按钮
And 系统颁发的认证令牌在24小时内有效

## 边界情况

### 场景：使用格式错误的邮箱登录
When 用户在邮箱输入框中填写 "not-an-email"
And 在密码输入框中填写任意内容
And 点击登录按钮
Then 页面不跳转，仍在登录页面
And 邮箱输入框下方显示红色提示文字"邮箱格式无效"
And 登录按钮恢复为可点击状态

## 错误条件

### 场景：系统服务不可用时登录
Given 系统所依赖的数据库服务不可用
When 用户在邮箱输入框中填写有效邮箱 "alice@example.com"
And 在密码输入框中填写正确密码 "SecurePass123!"
And 点击登录按钮
Then 页面不跳转，仍在登录页面
And 页面顶部显示黄色横幅提示"服务暂时不可用，请稍后重试"
And 不显示任何内部错误信息或技术细节
```
